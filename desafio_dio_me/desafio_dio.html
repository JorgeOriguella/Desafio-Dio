História da Internet

A hitória da internet começa em meio a Guerra Fria, que ocorreu entre EUA e URSS. Em 1969, os Estados Unidos com medo de um ataque da União Soviética às suas bases militares, criaram a ARPANET para que a troca de dados fosse mais segura, e os dados não ficassem centralizados em um único lugar, correndo risco de em um ateque às suas bases militares, perder todas as suas informações.

Após a tensão entre os dois países esfriar, na década de 1970, a ARPANET foi aberta para localidades não militares. Desta forma, a rede ficou muito grande, surgiram novos protocolos, até o cientista Tim Beners-Lee criar a World Wide Web, o WWW e o protocolo HTTP, que contribuiram para chegar aos dias de hoje.

O que são Clients?

Os Clients ou traduzindo, os Clientes, são desktops, celulares, tablets entre outros dispositivos que fazem a requisição de serviços ou recursos de um servidor. É a parte que interage com o usuário, como os navegadores, aplicativos de e-mail e de mensagens, programas de transferência de arquivos etc.

O cliente envia solicitações ao servidor, que pode ser uma solicitação para carregar uma página web, enviar um e-mail ou acessar um arquivo. O servidor processa a solicitação e envia a resposta de volta para o cliente, que então exibe a informação ou realiza a ação solicitada.

O que são Servers?

Os Servers ou servidores, fica no lado oposto aos clients, eles respondem as solicitações feitas pelos clients, como dizemos anteriormente esses serviços podem incluir armazenamento de dados, hospedagem de sites, gerenciamento de e-mails, processamento de aplicativos, entre outros. Em termos simples, um servidor atende às solicitações de clientes, fornecendo-lhes os recursos ou informações que eles precisam.

Alguns exemplos de servidores são: Servidor Web, Servidor de Banco de Dados, Servidor de Arquivos, Servidor de E-mail, Servidor de Aplicações entre outros.

Os servidores podem ser:

Servidores Locais, que são servidores físicos instalados dentro da própria empresa, geralmente em uma sala de servidores.

Servidores em Nuvem, que são servidores virtuais hospedados em data centers espalhados pelo mundo.

Data Centers, que são locais com infraestrutura especializada para abrigar servidores, com sistemas de refrigeração, energia e segurança.

HTML Semântico

Mas o que é HTML semântico?

HTML significa Hypertext Markup Language, que nada mais é que o “esqueleto” de uma página.

Semântica é o estudo do significado e interpretação de palavras.

O HTML semântico é a forma de deixar o site com suas informações bem explicadas e compreensíveis para o computador, ajudando até mesmo em sua busca no Google, leitores de tela e facilitando o entendimento de leitores de acessibilidade.

Elementos semânticos

Temos várias tags consideradas semânticas, por exemplo, tags para estruturar uma página web, como:

<header>, <nav>, <main>, <section>, <article>, <aside>, <footer>. Semanticamente falando, essas tags, ou parte delas, fazem sentido juntas.

Elementos não semânticos

Diferentemente das tags citadas acima, <div>s, <span>s, não diz nada sobre seu conteúdo, é ideal apenas para estilização.

Também podemos incluir outras tags como <b>, que especifica texto em negrito sem nenhuma importância extra, ao contrário do <strong>, que é usada para definir texto com grande importância.

Web Scraping

Web scraping é a prática de extrair dados de websites de forma automatizada, geralmente usando programas de computador. Em vez de copiar e colar manualmente o conteúdo de um site, o web scraping utiliza ferramentas que simulam a navegação humana, extraindo informações específicas, como texto, imagens, preços ou qualquer tipo de dado disponível.

O Web Scraping tem diversas finalidades, entre elas são: Monitoramento de preços, pesquisa de mercado, monitoramento de redes sociais, até mesmo para desenvolvimento de inteligência artificial.

Mas em termos jurídicos, é legal fazer o web scraping?

É importante ressaltar que o web scraping pode ter questões legais envolvidas, especialmente quando se trata de dados não disponíveis publicamente ou quando o scraping é feito de forma excessiva, sobrecarregando o servidor do site. É fundamental verificar os termos de serviço do site e respeitar as leis de privacidade de dados.

Acessibilidade

A Acessibilidade em sites refere-se à prática de projetar e desenvolver páginas web de forma que pessoas com deficiência possam percebê-las, entendê-las, navegá-las e interagir com elas de maneira eficaz. Isso inclui garantir que sites sejam acessíveis para pessoas com deficiência visual, auditiva, motora e cognitiva, entre outras.

Isso pode envolver a implementação de recursos como: Legendas e transcrições para vídeos, tag <track>, texto alternativo para imagens, atributo alt="", navegação por teclado, alto contraste e tamanho de fonte ajustável, linguagem clara e simples entre outros. Desta forma garante a inclusão, o direito de acessibilidade, gerando benefícios para todos e boas práticas.
